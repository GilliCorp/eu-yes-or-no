## 🇮🇸 Íslenska

### Um hvað snýst þetta?

Þessi vefsíða útskýrir, á mannamáli og með heimildum, hvað myndi raunverulega breytast fyrir Ísland við inngöngu í Evrópusambandið (ESB).

Lykilatriðið: Ísland er nú þegar um **tveir þriðju „inni"** í ESB gegnum EES-samninginn og Schengen. Spurningin er því ekki „ESB eða ekkert" — heldur **hvað breytist** ef við göngum alla leið. Hver kafli ber saman stöðuna í dag (undir EES) við stöðuna sem fullt aðildarríki.

Síðan tekur **enga afstöðu**. Hún reynir ekki að sannfæra þig um eitt eða neitt — hún safnar staðreyndum og heimildum svo þú getir tekið upplýsta ákvörðun sjálf/ur.

## 🇬🇧 English

### What is this?

This site explains, in plain language and with sources, what would actually change for Iceland if it joined the European Union (EU).

The key reframe: Iceland is already about **two-thirds "in"** the EU through the EEA Agreement and Schengen. So the question isn't "EU or nothing" — it's **what changes** if we go all the way. Every chapter compares the situation today (under the EEA) with the situation as a full member.

The site takes **no side**. It doesn't try to convince you of anything — it gathers the facts and sources so you can make an informed decision yourself.

### How neutrality is enforced

- **Every single claim must cite a source** — or the site fails to build (it's baked into the code).
- **All text exists in both Icelandic and English** — same rule.
- All content is open here on GitHub, and the edit history is visible.
- The text is still **AI-drafted and awaiting native review** — stated openly.
- More: [Methodology](https://gillicorp.github.io/eu-yes-or-no/en/methodology/).

### Found an error? Want to challenge a claim?

Great — that's exactly the point. **Open an issue:** [New issue →](https://github.com/GilliCorp/eu-yes-or-no/issues/new)

To make it as useful as possible, include:

1. **Which page/chapter and which claim** you're challenging (paste the text).
2. **Why** you think it's wrong or misleading.
3. **Evidence** for your case — ideally a primary source (law, court ruling, statistics, official document). A link is best.

Every challenge gets reviewed. If the evidence holds up, the page is updated with corrected or new information — that's how it evolves, honestly and with sources.

## 🛠️ For developers

A static [Astro](https://astro.build) site (zero-JS, content collections), bilingual via a small i18n table. To run locally:

```sh
npm install      # install dependencies
npm run dev      # local dev server at localhost:4321
npm run build    # production build to ./dist/
npm test         # unit tests (loan / grocery-basket math)
```

Chapter content lives in `src/content/dossiers/*.yaml`; the Start-here and Methodology pages in `src/data/`. The Zod schema in `src/content.config.ts` enforces the "every claim needs a source" and "bilingual" rules at build time — break either and the build fails. Auto-deploys to GitHub Pages on push to `main`.
